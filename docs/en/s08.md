# s08 — In-Process MCP Server

## Problem

Users want to expose Go functions as tools Claude can call. "Out-of-process" MCP servers (separate binaries) are expensive: extra IPC, deployment, lifecycle. In-process means the tool runs in the same Go program as the agent — no fork, no socket.

## Solution

A `Server` that maps tool names to handlers. Each `Tool` carries Name, Description, JSON-Schema, and a Handler func. A `SchemaFromStruct` reflection helper builds the schema from struct tags.

## How It Works

Python's `@tool` decorator and `_python_type_to_json_schema` reflect on the function signature at registration time. Go doesn't have decorators; instead, the user passes `mcp.SchemaFromStruct(MyArgs{})` to get a JSON-Schema map from struct tags. The `Server.Call(name, args)` dispatches by name and returns `CallResult` (text content + error flag).

## What Changed

- New module `agents/s08-mcp`.
- `server.go`: `Server`, `Tool`, `Handler`, `CallResult`, `SchemaFromStruct`, `Text` helper.
- Five tests: register+call, unknown tool, listing order, schema derivation, Text helper.

## Try It


- `src/claude_agent_sdk/__init__.py:create_sdk_mcp_server` — the constructor. Note how `_python_type_to_json_schema` walks the input dataclass; that's the part our `SchemaFromStruct` replicates.
- `src/claude_agent_sdk/__init__.py:tool` — the decorator. Python stamps name/description/schema onto a callable; Go uses an explicit struct.
