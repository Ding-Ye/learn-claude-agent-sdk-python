# s05 — Streaming Client

## Problem

`Query()` spawns + tears down per call. For a chat app, you pay startup cost on every turn. You also can't react to a response and follow up without losing context.

## Solution

`Client` keeps the subprocess alive. `Connect` once, `Send(prompt)` per turn, consume from one shared `Recv()` channel. `Close` ends the session.

## How It Works

A single goroutine pumps stdout into a channel for the entire client lifetime. Each `Send` writes one envelope line. The client doesn't track turn boundaries internally — callers loop on `Recv()` until they see a `ResultMessage`, then send the next prompt.

## What Changed

- New module `agents/s05-client`.
- `client.go`: `New`, `Connect`, `Send`, `Recv`, `Err`, `Close`.
- A streaming fake `cli/main.go` that handles **multiple** prompt lines.

## Try It

