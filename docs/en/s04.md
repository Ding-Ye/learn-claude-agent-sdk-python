# s04 — Query (one-shot)

## Problem

Composition: how do you stitch the transport (s02) and the parser (s03) into a single function that says "send this prompt, give me back the messages"?

## Solution

`Query(ctx, prompt, opts) (<-chan Message, <-chan error)`. Internally: spawn process → write JSON envelope → close stdin → drain stdout → parse → push to channel → stop on `ResultMessage`.

## How It Works

We pass options as a JSON envelope on the first stdin line (matches how the real CLI accepts config). Closing stdin tells the CLI "no more turns". A goroutine reads stdout and feeds the channel; the moment we see a `ResultMessage`, we close the channels and `cmd.Wait()`.

## What Changed

- New module `agents/s04-query` with all three previous concerns (types, transport, parser) inlined.
- Public surface: `Query`, `Options`.
