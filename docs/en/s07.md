# s07 — Hooks

## Problem

User code wants to interpose on the agent loop: log a tool call, deny it, rewrite its args, decide to stop a turn early. The SDK has lots of these "hook points"; we don't want to bake one method into the agent struct per hook.

## Solution

A `Registry` of `Matcher{Event, Tools, Callbacks}`. The CLI emits a `hook_event` with a name like `"PreToolUse"`; the registry runs every matching callback. A callback's `Output{Continue, StopReason, Decision, Replace}` carries any short-circuit, deny, or rewrite info back.

## How It Works

`Dispatch` walks the matcher list, filters by event name and optional tool name list, runs each callback in registration order. If a callback returns `Continue=false`, the chain stops — matches upstream's "stop the turn now" semantics.

## What Changed

- New module `agents/s07-hooks`.
- `hooks.go`: `Event`, `Output`, `Callback`, `Matcher`, `Registry`.
- Four tests for routing, tool filter, short-circuit, and error propagation.
