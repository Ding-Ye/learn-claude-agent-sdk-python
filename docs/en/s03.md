# s03 — Parser

## Problem

The transport hands us strings. Each string is supposed to be a JSON object describing one message — but with polymorphic content (an `assistant` message can contain a `text` block followed by a `tool_use` block followed by a `thinking` block). We need to land it in typed Go values without a 3-page switch statement.

## Solution

`Parse(line []byte) (Message, error)`. A two-step approach: peek at the `type` field with a tiny anonymous struct, then dispatch into a per-variant decoder. The same trick repeats for content blocks. Unknown content blocks are silently dropped (mirrors upstream's forward-compat policy).

## How It Works

`json.RawMessage` is the trick that makes the two-step work: we leave the bytes uninterpreted while peeking, then re-decode into the right struct. For content blocks, we peek their `type` and skip ones we don't recognize so a new CLI version doesn't crash an old SDK.

## What Changed

- New module `agents/s03-parser`.
- `parser.go` re-defines the message + block types (no cross-session imports), provides `Parse` and a small `ParseError`.
- `parser_test.go` covers all four message types + the unknown-block-skip behavior.
