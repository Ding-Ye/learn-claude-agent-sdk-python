# s01 — Types

## Problem

The SDK sits between two foreign worlds: Python user code on one side, the `claude` CLI's JSON stream on the other. Before we can spawn anything or parse anything, we need a *shape*: what does a "Message" look like in our language? What's a "ContentBlock"?

## Solution

Define a sealed `Message` interface with four implementations (User / Assistant / System / Result), and a sealed `ContentBlock` interface with four implementations (Text / Thinking / ToolUse / ToolResult). Add a `PermissionMode` constants block. That's the whole data model.

## How It Works

Go has no tagged sums, so we pick the **sealed interface** pattern: each variant defines a tiny `messageKind() string` method whose name is unexported, so external packages can't add new variants. JSON serialization rides a `type` discriminator field, just like the wire protocol.

- `EncodeBlock` / `DecodeBlock` round-trip a `ContentBlock` through wire JSON.
