# s02 — Transport

## Problem

The CLI is a separate process. The SDK has to spawn it, write a prompt to its stdin, and read a stream of JSON lines from its stdout — without leaking processes when the parent crashes, and without livelocking on shutdown.

## Solution

A `SubprocessTransport` that owns an `os/exec.Cmd`. `Start` spawns + wires pipes. A goroutine pumps stdout into a buffered string channel. `Send` writes one line at a time. `Close` half-closes stdin, waits 2s for graceful exit, then `SIGKILL`s.

## How It Works

Stdin pipe → user writes JSON envelope. Stdout pipe → scanner with 1MB buffer (matches upstream) → channel. Stderr discarded for now (a real port would route to a logger). The mutex guards `Start`/`Close` so a `Start` race during shutdown becomes an error instead of a deadlock.
