# s01 — 类型

## Problem

SDK 夹在两个外部世界之间：一边是用户的 Python（或 Go）代码，另一边是 `claude` CLI 吐出的 JSON 流。在能 spawn 任何东西、parse 任何东西之前，得先有"形"：在我们的语言里，"Message"长什么样？"ContentBlock"长什么样？

## Solution

定义一个封闭的 `Message` 接口加 4 个实现（User / Assistant / System / Result），再定义一个封闭的 `ContentBlock` 接口加 4 个实现（Text / Thinking / ToolUse / ToolResult），加一组 `PermissionMode` 常量。整个数据模型就完了。

## How It Works

Go 没有 tagged sum，所以走 **sealed interface** 模式：每个 variant 实现一个 `messageKind() string` 方法，方法名小写不导出，外部包加不了新 variant。JSON 序列化用 `type` 字段做判别，跟线协议一致。

## What Changed（相对"空"——这是地基）

- 新建 `agents/s01-types/types.go` 和 `types_test.go`。
- `EncodeBlock` / `DecodeBlock` 让 `ContentBlock` 能在 wire JSON 和 Go struct 之间往返。
