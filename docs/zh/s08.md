# s08 — 进程内 MCP 服务器

## Problem

用户想把 Go 函数暴露成 Claude 能调的工具。"进程外" MCP（独立二进制）贵：额外 IPC、部署、生命周期。"进程内"指工具跟 agent 在同一个 Go 程序里——不 fork、不开 socket。

## Solution

`Server` 把工具名映射到 handler。每个 `Tool` 带 Name、Description、JSON Schema、Handler。`SchemaFromStruct` 反射 helper 从 struct tag 生成 schema。

## How It Works

Python 的 `@tool` 装饰器加 `_python_type_to_json_schema` 在注册时反射函数签名。Go 没装饰器；让用户传 `mcp.SchemaFromStruct(MyArgs{})` 得到 JSON Schema map。`Server.Call(name, args)` 按名分发，返回 `CallResult`（文本内容 + error 标志）。

## What Changed

- 新模块 `agents/s08-mcp`。
- `server.go`：`Server`、`Tool`、`Handler`、`CallResult`、`SchemaFromStruct`、`Text` helper。
- 五个测试：注册+调用、未知工具、列表排序、schema 反射、Text helper。
