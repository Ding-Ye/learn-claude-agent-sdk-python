# s05 — 流式客户端

## Problem

`Query()` 每次都起进程、拆进程。聊天场景每轮都要付冷启动成本。而且想根据上一轮回复决定下一轮 prompt 也很难。

## Solution

`Client` 保持子进程活着。`Connect` 一次，每轮 `Send(prompt)`，统一从 `Recv()` channel 读。`Close` 关 session。

## How It Works

一个 goroutine 把 stdout 抽进 channel，整个 client 生命周期共用。每次 `Send` 写一行 envelope。client 自己不管轮次边界——caller 在 `Recv()` 上循环直到看到 `ResultMessage`，再发下一个 prompt。

## What Changed

- 新模块 `agents/s05-client`。
- `client.go`：`New`、`Connect`、`Send`、`Recv`、`Err`、`Close`。
- 一个流式假 CLI `cli/main.go`，能处理**多行** prompt。

## Try It

```
cd agents/s05-client
go test ./...
```
