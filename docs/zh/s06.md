# s06 — 工具权限

## Problem

CLI 要调一个工具。该不该让它调？"该"可能因为预先 allowlist、因为 mode、因为用户回调。"不该"可能因为 blocklist、因为默认拒。需要一条确定、可调试的决策链。

## Solution

一个 `Policy{Allowed, Disallowed, Mode, CanUseTool}` 加一个 `Evaluate(ctx, name, input)`。六步链：disallow-list → allow-list → bypass → dontAsk → callback → 默认拒。

## How It Works

顺序很重要。disallow 赢 allow，是为了护栏团队能放心 blocklist 一个工具，不用怕被 allow-list 抵消。callback 是**最后**那道编程门——只要前面 list/mode 没决定，用户就被问。callback 收到 `(name, input)` 后可以重写 input、错位允许（updated_input）、或拒绝。

## What Changed

- 新模块 `agents/s06-tools-permissions`。
- `perm.go`：`Policy`、`Decision`、`CanUseTool`、`Policy.Evaluate`。
- 九个测试覆盖决策链每一支。
