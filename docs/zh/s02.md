# s02 — Transport

## Problem

CLI 是另一个进程。SDK 要起它、把 prompt 写进它的 stdin、再读 stdout 上的 JSON 行流——并且：父进程崩了不能漏子进程，关闭流程不能死锁。

## Solution

`SubprocessTransport` 持有一个 `os/exec.Cmd`。`Start` 起进程并接管 pipe。goroutine 把 stdout 抽进一个 buffered string channel。`Send` 写一行。`Close` 半关 stdin，等 2 秒，超时则 SIGKILL。

## How It Works

stdin pipe ← 用户写的 JSON envelope。stdout pipe → scanner（1MB buffer，跟上游一致）→ channel。stderr 先丢掉（真的端口会接到 logger）。一把 mutex 守 `Start`/`Close`，让关停时的 `Start` 竞争变报错而不是死锁。
