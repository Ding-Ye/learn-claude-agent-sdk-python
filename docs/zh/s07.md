# s07 — Hooks

## Problem

用户代码想介入 agent loop：记一条工具调用、拒它、改它的参数、提前停掉当前轮。SDK 这种"hook 点"很多；不想给 agent struct 加 N 个方法。

## Solution

一个 `Matcher{Event, Tools, Callbacks}` 列表组成的 `Registry`。CLI 吐 `hook_event`（名字像 `"PreToolUse"`），registry 跑所有匹配的 callback。callback 的 `Output{Continue, StopReason, Decision, Replace}` 把短路、拒绝、改写信息带回来。

## How It Works

`Dispatch` 遍历 matcher，按事件名 + 可选工具名过滤，按注册顺序跑每个 callback。callback 返回 `Continue=false` 整条链停——跟上游的"现在就停止本轮"语义一致。

## What Changed

- 新模块 `agents/s07-hooks`。
- `hooks.go`：`Event`、`Output`、`Callback`、`Matcher`、`Registry`。
- 四个测试：路由、工具过滤、短路、错误传播。
